# モバイルアプリの画面構成作成とナビゲーションの改善

## 概要
モバイルアプリのディレクトリ構造を整理し、Expo Go でも快適に開発が進められるようにデバッグ用のナビゲーション機能を実装しました。また、プロジェクト構成を最新の Expo 推奨設定に合わせるためのリファクタリングを行いました。

## 実施内容

### 1. 画面構造の構築 (Expo Router)
- `(auth)` グループ: `login`, `signup`（`register` から改名）のプレースホルダー画面を作成。
- `(main)` グループ: `notes`（一覧）、`notes/[id]`（詳細）のプレースホルダー画面を作成し、タブナビゲーションを設定。

### 2. 開発用ナビゲーションの実装
- `expo-dev-menu` が Expo Go で動作しない問題を考慮し、**独自のフローティング開発メニュー**を `features/home` に実装しました。
- 開発モード (`__DEV__`) 時のみ画面右下に表示され、各画面へワンタップで遷移可能です。

### 3. リファクタリング
- **ディレクトリ構成の整理**: `app/index.tsx` のロジックを `features/home/components/HomeScreen.tsx` へ分離し、ディレクトリ間の責務を明確化。
- **設定の適正化**: `app.json` から `"newArchEnabled": false` を削除し、最新の React Native Architecture をサポートするように調整。

### 4. その他
- 認証周りの命名を `register` から `signup` へ統一。
- 不要になった依存関係の整理。

## プレビュー
- 実機/シミュレータの画面右下にある「メニューボタン」から、作成した各画面へのアクセスが可能です。
